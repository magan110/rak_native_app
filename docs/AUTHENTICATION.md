# Authentication & Authorization

## Overview

The RAK App implements a multi-method authentication system with role-based access control (RBAC). Users can authenticate via **password** or **OTP**, and the app supports **auto-login** for returning users.

---

## Authentication Methods

### 1. Password Authentication

**Flow**: `LoginWithPasswordScreen` → `AuthService.authenticateUser()`

```
User enters User ID + Password
        │
        ▼
AuthService.authenticateUser(userID, password, appRegId)
        │
        ▼
POST /api/Auth/login  ──► Server validates
        │
        ▼
LoginResponse { msg, UserData }
        │
        ▼
AuthManager.setUser(userData)  ──► Route to role-specific home
```

### 2. OTP Authentication

**Flow**: `LoginScreenWithOtp` → `SmsUaeService` → `AuthService`

```
User enters mobile number
        │
        ▼
SmsUaeService.verifyMobile(mobile)  ──► Check if registered
        │
        ▼
SmsUaeService.sendSms(mobile, otp)  ──► Send OTP via UAE SMS gateway
        │
        ▼
User enters received OTP
        │
        ▼
AuthService.loginWithOtp(mobile, otp)  ──► Validate & authenticate
```

### 3. Auto-Login

**Flow**: `SplashScreen` → `StorageService` → `AuthService.autoLogin()`

```
App Launch (SplashScreen)
        │
        ▼
StorageService.canAutoLogin()  ──► Check for stored appRegId
        │
        ├── Yes ──► AuthService.autoLogin()
        │               │
        │               ▼
        │           POST /api/Auth/auto-login { appRegId }
        │               │
        │               ▼
        │           Route to role-specific home
        │
        └── No  ──► Navigate to Login Screen
```

## Session Management

### App Registration ID (`appRegId`)

Each device/installation gets a unique `appRegId` generated by `StorageService.generateAppRegId()`. This ID is:
- Created on first authentication
- Stored locally via `SharedPreferences`
- Sent with every login request
- Used for auto-login on subsequent app opens
- Cleared on logout

### Auto-Login Hash

For enhanced security, the app generates a secure hash combining:
- User ID
- User type (painter/contractor)
- Device ID (optional)
- Timestamp
- Random data

This hash is stored locally and validated against stored user data on app launch.

### Storage Keys

| Key | Type | Purpose |
|---|---|---|
| `user_token` | String | Authentication token |
| `user_data` | String | Serialized user data |
| `app_reg_id` | String | Device registration ID |
| `user_id` | String | Remembered user ID |
| `remember_me` | Bool | Remember me preference |
| `auto_login_hash` | String | Auto-login verification hash |
| `user_type` | String | User type (painter/contractor) |

## Role-Based Access Control

### User Data Model

```dart
class UserData {
  final String emplName;    // Employee name
  final String areaCode;    // Assigned area
  final List<String> roles; // Assigned roles
  final List<String> pages; // Accessible pages
  final String? userID;
  final String? appRegId;
}
```

### Access Check Methods

| Method | Description |
|---|---|
| `hasRole(String role)` | Exact role match |
| `hasPage(String page)` | Exact page access check |
| `hasAnyRole(List<String> roles)` | Match any of multiple roles |
| `hasAnyPage(List<String> pages)` | Match any of multiple pages |

### User Types & Home Screens

## Logout

```dart
// Client-side
AuthService.logout()
  → StorageService.clearAllAuthDataEnhanced()
  → AuthManager.clearUser()

// Server-side
AuthService.logoutFromServer()
  → POST /api/Auth/logout { appRegId }
  → StorageService.clearAllAuthDataEnhanced()
  → AuthManager.clearUser()

// Navigation
AppRouter.logout(context)
  → AuthService.logout()
  → Navigate to login screen
```

## Security

### SSL Certificate Pinning

All API communication uses SSL pinning via `SslHttpClient`:
- PEM certificate bundled at `assets/cert/rak_cer.pem`
- Domain whitelist: `birlawhite.com`, `qa.birlawhite.com`
- Fallback to permissive validation if certificate loading fails

### OTP Security

| Setting | Value |
|---|---|
| OTP TTL | 5 minutes |
| Resend cooldown | 30 seconds |
| Message template | `"Dear User Your One time password OTP is : {otp} Birla White (RAK)"` |
| Delivery | UAE SMS gateway via `SmsUaeService` |

### HTTP Headers

The app sends browser-like HTTP headers to bypass WAF/security systems, including:
- Chrome mobile `User-Agent`
- CORS headers (`Origin`, `Sec-Fetch-*`)
- Cache-busting (`Cache-Control: no-cache`)
- `X-Requested-With: XMLHttpRequest`
